Imagenes generadas utilizando API Canvas 2D Context y funciones matemáticas

Desde que se comenzó a hacer público todo lo relacionado con HTML5 y sus potencialidades, una de las cosas que me impresionó fue la API Canvas 2D Context que viene incorporada como algo novedoso y muy interesante. Y es que realmente lo que se puede hacer con esta API y la integración que permite con imágenes y videos es indudablemente muy prometedor, muchos ya expresan que estas tecnologías darán un vuelco a la web, como por ejemplo, que la reproducción de video de manera nativa en los navegadores -sin usar plugins de terceros- será el fin de los reproductores basados en Flash.

En fin, no voy a dar una disertación sobre ello, porque no soy un experto ni nada por el estilo, solo voy dar una pequeña explicación sobre lo que hace esta "aplicación web?", si no crees que te haga falta leer o simplemente no quieres leer esto, pues te lo saltas y ahorras tiempo.

Esta aplicación utiliza la API Canvas 2D Context y varias funciones del objeto Math, para generar imágenes al vuelo que son mostrados en un elemento "canvas", permitiendo almacenar la imagen generada de manera temporal mediante el botón "Tomar instantánea"; permitiendo luego listarlas, eliminarlas o guardarlas en el disco si te a gustado, esto se logra mediante los 6 botones a la derecha del elemento canvas y debajo del elemento imagen, que mostrara una instantánea tomada con anterioridad.

Aclaración 1: Esto lo hice porque aunque en Mozilla Firefox guardar la imagen directamente desde el elemento canvas es muy sencillo -basta con dar clic derecho y luego "Guarda imagen como"- Chrome y Opera no tienen esta posibilidad, además de agregar cierta funcionabilidad a la aplicación.

Aclaración 2: Esta aplicación no tiene el objetivo de ser la mejor, ni la más rápida, ni la más optimizada, ni la más "X" en cuanto a mostrar las potencialidades y demás características de estas tecnologías; aun así, las críticas, sugerencias y opiniones son bienvenidas.

Debajo de los botones que se utilizan para las administración de las instantáneas tomadas, está una lista de nombres o descripciones de imágenes que se generan al vuelo, una vez seleccionadas -luego activar el botón "Iniciar" para comenzar la animación- encabezada por el subtitulo "Imágenes predefinidas:", cada elemento de la lista lo que describe son los valores o datos que irán en cada uno de los elementos situados debajo del elemento canvas (XMid, YMid, X, Y, etc.), los cuales se evaluarán mediante la función eval- como código JavaScript y que será utilizado para la generación de la imagen correspondiente a estos valores, estas son muestras ya predefinidas. El botón "Animación a JSON" permite obtener en dicho formato los parámetros y los valores que describen la misma.

Si mueven el puntero del ratón por encima de algunos de los botones de control de animación (Iniciar, Detener, Limpiar, Tomar instantánea) y las lista de "Imágenes predefinidas", podrán observar que se mostrara un pequeño cuadro de información (tooltip) que contendrá las teclas de acceso para cada uno de estos elementos, con ellas se pueden ejecutar las acciones correspondientes a cada uno; esto es útil ya que si se activa la "modo de pantalla completa", el elemento canvas se ajustará para ocupar toda la pantalla, permitiendo ver la animación a pantalla completa, así se podrá iniciar, detener, tomar una instantánea o cambiar hacia otra imagen predefinida, sin tener que salir del "modo de pantalla completa" o tener que utilizar el mouse.

Descripción de los campos:

  • XMid: Representa la coordenada respecto al eje X, desde donde se comenzará a imprimir la imagen en el elemento canvas.
  • YMid: Representa la coordenada respecto al eje Y, desde donde se comenzará a imprimir la imagen en el elemento canvas.
  • X: Expresión que se evaluará en cada interacción que devolverá la coordenada respecto al eje X del próximo punto a imprimir.
  • Y: Expresión que se evaluará en cada interacción que devolverá la coordenada respecto al eje Y del próximo punto a imprimir.
  • Inc angle: Valor que se utilizará para el incremento en cada interacción del ángulo utilizado en la generación de la imagen.
  • Radio: Valor de inicio establecido para el radio -referente a circunferencia- utilizado en la generación de la imagen.
  • Inc radio: Valor que se utilizará para el incremento en cada interacción del radio.
  • Desvanecimiento: Habilita o deshabilita el efecto de desvanecimiento de la imagen.

Estos pueden ser editados con el objetivo de experimentar y generar nuevas imágenes. Es de señalar que la ejecución es mucho más rápida en Chrome y Opera que en Firefox.
